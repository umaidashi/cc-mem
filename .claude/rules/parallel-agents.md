# 並行エージェント活用パターン

このプロジェクトでは独立した作業を Agent tool で並行実行する。

## パターン

### 独立モジュール開発
依存関係のないモジュール（例: db, chunker, embedder）は別チームに分けて並行実装。

### TDD の Red/Green 並行
- テスト作成（QAチーム）と実装（実装チーム）を並行で走らせられるケース:
  - インターフェースが明確に定義済み
  - テストが先に完成する保証がある場合
- 不明な場合はテスト完成を待ってから実装

### ドキュメント並行更新
README と docs/ の更新は独立しているので並行可能。

## 注意

- 同一ファイルを複数エージェントが編集しないこと
- エージェントの出力は CEO（メインコンテキスト）が検証すること
- subagent は MCP tool やコンテキストを圧迫する処理の委譲にも活用
