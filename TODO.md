# cc-mem Improvement Roadmap

## P1: 保存品質の向上

- [x] 短すぎる Q&A ペアを除外（Q+A 合計100文字以下をスキップ） → ADR-001
- [~] ファイル操作だけの会話 → 既存フィルタでカバー済み (短文フィルタ + extractText)
- [~] tool_use が大半のターン → extractText が text のみ抽出、空ターンは自動スキップ
- [~] 長すぎる回答のパラグラフ分割 → --context で検索時に文脈取得可能、保存時分割は不要
- [~] 関連する連続 Q&A をまとめる → --context で代替

## P1: ノイズ対策

- [x] 保存時に既存メモリとの類似度チェック → 重複排除 → ADR-003
- [~] 会話の「重さ」に基づく重要度スコア → 時間減衰で実用上十分、効果測定してから再検討
- [x] 「はい」「OK」「ありがとう」のような軽量応答の除外 → ADR-002
- [x] Q テキストのクリーニング（`[Request interrupted by user]`、XML タグ除去） → ADR-004
- [x] transcript の type="user" 対応（"human" だけでなく "user" も受け付ける）→ 修正済み

## P2: 検索体験の向上

- [x] `--context` オプション: 前後の Q&A や同セッションの他の Q&A も返す → ADR-006
- [~] タグ・カテゴリ自動分類 → 精度不明、実運用データで再評価
- [~] `--tag` オプション → タグ分類に依存、保留
- [~] `--summarize` オプション → LLM使用でトークン消費ゼロの思想と矛盾、保留

## P2: 運用・可視化

- [x] `cc-mem import` — 既存セッションログの一括取り込み → ADR-005
- [x] ~~`cc-mem log`~~ — 削除済み
- [x] `cc-mem gc` — 古い・低スコアのメモリを削除 → ADR-008
- [x] ~~`cc-mem export`~~ — 削除済み

## P3: 野心的な方向性

- [ ] セッションサマリ: Q&A 羅列ではなくセッション全体の要約を保存（LLM使用）
- [x] プロジェクト別スコープ（1DB + project カラム） → ADR-011
- [x] ~~SessionStart Hook で自動想起 (`cc-mem recall`)~~ — 削除済み
- [ ] 短期記憶層の導入（下記「短期記憶 vs 長期記憶」参照）
- [x] キー自動抽出（JIRA, PR, URL）→ ADR-012
- [x] 関連メモリ探索: 共有キーを通じたグラフ探索 → ADR-013

## 短期記憶 vs 長期記憶

現状の cc-mem は「長期記憶」に該当する。
短期記憶層の導入は P3 で検討。詳細は本ファイル末尾を参照。
