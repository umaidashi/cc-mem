# cc-mem Improvement Roadmap

## P1: 保存品質の向上

- [x] 短すぎる Q&A ペアを除外（Q+A 合計100文字以下をスキップ） → ADR-001
- [ ] ファイル操作だけの会話（コード出力が大半）を除外
- [ ] tool_use が大半のターンをフィルタリング
- [ ] 長すぎる回答のパラグラフ分割
- [ ] 関連する連続 Q&A をまとめるオプション

## P1: ノイズ対策

- [x] 保存時に既存メモリとの類似度チェック → 重複排除 → ADR-003
- [ ] 会話の「重さ」に基づく重要度スコア（長い議論 > 短い応答）
- [x] 「はい」「OK」「ありがとう」のような軽量応答の除外 → ADR-002

## P2: 検索体験の向上

- [ ] `--context` オプション: 前後の Q&A や同セッションの他の Q&A も返す
- [ ] タグ・カテゴリ自動分類（design, debug, config, discussion）
- [ ] `--tag` オプションで絞り込み検索
- [ ] `--summarize` オプション: 検索結果を LLM で要約（オプトイン）

## P2: 運用・可視化

- [ ] `cc-mem log` — 直近N セッションの保存履歴を表示
- [ ] `cc-mem gc` — 古い・低スコアのメモリを削除
- [ ] `cc-mem export/import` — JSON エクスポート・インポート

## P3: 野心的な方向性

- [ ] セッションサマリ: Q&A 羅列ではなくセッション全体の要約を保存（LLM使用）
- [ ] プロジェクト別 DB vs グローバル DB の選択
- [ ] SessionStart Hook で自動想起 → system prompt に注入
- [ ] 短期記憶層の導入（下記「短期記憶 vs 長期記憶」参照）

## 短期記憶 vs 長期記憶

現状の cc-mem は「長期記憶」に該当する。
短期記憶層の導入は P3 で検討。詳細は本ファイル末尾を参照。
