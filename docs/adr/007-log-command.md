# ADR-007: cc-mem log コマンド

## Status
Accepted

## Context
保存された記憶の概要を確認する手段がない。stats は全体の数字だけ、search はクエリが必要。「最近何が保存されたか」「どのセッションが入っているか」をサッと見たい。

## Decision
`cc-mem log` コマンドを追加する。

### 基本動作
- 直近 N セッション（デフォルト 10）の概要を表示
- 各セッションの保存日時、メモリ件数、最初の Q を表示

### オプション
- `--last N`: 表示セッション数（デフォルト 10）
- `--all`: 全セッション表示

### 出力例
```
cc-mem log (10 sessions)
────────────────────────────
sess_20260324_110910 | 2026-03-24 | 12 memories
  → ReactのuseStateとは？
  → SQLiteのWALモードについて教えて
  → ...

import_ee583385 | 2026-03-24 | 9 memories
  → チケットに取り組んでほしい
  → PRのコメントを確認
  → ...
```

各セッションの最初の3件の Q を表示（概要把握に十分）。

## Acceptance Criteria
1. `cc-mem log` でデフォルト10セッション分の概要を表示
2. `--last 5` で5セッションに制限
3. 各セッションに日時、メモリ件数、最初3件のQが表示される
4. セッションは新しい順でソート
5. メモリが0件のセッションは表示されない（ありえないが念のため）

## Test Plan
テスト用DBに複数セッションのデータを直接INSERTし、log関数の出力を検証。

## Consequences
### メリット
- 蓄積状況を一目で確認できる
- 「このセッション消したい」等の運用判断ができる
### デメリット
- 軽微な追加（CLI + 1クエリ）
