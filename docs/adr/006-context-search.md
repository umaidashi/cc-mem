# ADR-006: --context 検索オプション

## Status
Accepted

## Context
現在の検索は個別の Q&A ペアを返すが、前後の文脈が欲しいケースが多い。例えば「設計判断」で検索してヒットした Q&A の前後で何を議論していたかがわからない。同じセッション内の他の Q&A を一緒に返せると、文脈を復元できる。

## Decision
`cc-mem search` に `--context` オプションを追加する。

### 動作
- `--context` が指定された場合、各検索結果に同セッションの前後 Q&A を付与する
- memories テーブルの id 順で前後 N 件（デフォルト 2）を取得
- 出力フォーマット: ヒットした Q&A の前後にインデントつきで表示

### 出力例
```
### #1 [score: 0.0164] (2026-03-24) session: sess_20260324_104234
  [context] Q: 前の質問...
  [context] A: 前の回答...
  **Q:** ヒットした質問
  **A:** ヒットした回答
  [context] Q: 後の質問...
  [context] A: 後の回答...
```

## Acceptance Criteria
1. `--context` なしの場合は既存動作と同じ
2. `--context` ありの場合、同セッションの前後2件の Q&A が表示される
3. セッションの先頭/末尾では前/後がない分だけ表示
4. 前後の Q&A のテキストも truncate される（Q: 80字、A: 150字）
5. 異なるセッションの Q&A は混ざらない

## Test Plan
1. 同セッション5件の Q&A を保存 → 3件目を検索でヒットさせる → 前後2件ずつ表示
2. セッション先頭の Q&A がヒット → 前はなし、後ろ2件のみ
3. --context なし → 従来と同じ出力
4. 異なるセッションの Q&A が混ざらないことを確認

## Consequences
### メリット
- 検索結果の文脈理解が大幅に向上
- 「あの議論の流れ」を復元できる
### デメリット
- 出力が長くなる（トークン消費増）
- 追加の DB クエリが必要
