# ADR-003: 重複排除

## Status
Accepted

## Context
同じ質問を繰り返し聞くと、似た Q&A が大量に蓄積する。検索結果の上位が似たような内容で埋まり、多様な情報にアクセスしにくくなる。特に定型的な開発作業（環境構築手順、よく使うコマンドなど）では同一内容の Q&A が何度も生成される。

## Decision
- 保存時に、新しいチャンクと既存メモリの類似度をチェックする
- embedding のコサイン類似度が閾値（デフォルト **0.95**）以上なら重複とみなしスキップする
- embedding がない場合（Ollama 未起動時など）は重複チェックをスキップし、そのまま保存する
- 閾値は環境変数 `CC_MEM_DEDUP_THRESHOLD` で変更可能とする
- 類似度の比較対象は Q のテキストに対する embedding とする（Q が同じで A が異なるケースを区別するため、Q+A の結合テキストで比較する）

## Acceptance Criteria
1. 同一の Q&A を2回保存しようとすると、2回目はスキップされる
2. 類似だが異なる Q&A（同じ質問で違う回答）は保存される
3. 閾値 0.95 でコサイン類似度を判定する（完全一致 ≒ 1.0 は確実にスキップ）
4. embedding なしの場合は重複チェックをスキップして保存する
5. 重複スキップ数が stderr に表示される
6. 既存データが 0 件の場合は全て保存される

## Test Plan
1. **完全一致テスト**: 同一の Q&A チャンクを2回保存処理に渡し、1回目は保存・2回目はスキップされることを確認する。SQLite 上のレコード数が1件であることを検証する
2. **類似だが異なるテスト**: Q=「Pythonのリスト内包表記とは？」に対して異なる A を持つ2つのチャンクを保存し、両方が保存されることを確認する
3. **閾値境界テスト**: コサイン類似度が 0.94（保存）と 0.96（スキップ）になるようなチャンクペアを用意し、閾値の境界動作を確認する
4. **embedding なしテスト**: Ollama が未起動の状態で重複チャンクを保存処理に渡し、重複チェックがスキップされて両方保存されることを確認する
5. **stderr 出力テスト**: 5件のチャンク（うち2件が重複）を処理し、stderr に「2件の重複をスキップ」相当のメッセージが出力されることを確認する
6. **空データベーステスト**: 既存データが 0 件の状態で複数チャンクを保存し、全件が保存されることを確認する
7. **環境変数テスト**: `CC_MEM_DEDUP_THRESHOLD=0.80` を設定し、類似度 0.80 以上のチャンクがスキップされることを確認する

## Consequences

### メリット
- 重複 Q&A の蓄積を防ぎ、検索結果の多様性が向上する
- SQLite のデータ量増加を抑制できる
- embedding なし時のフォールバック（チェックスキップ）により、Ollama 未起動でも保存機能が動作する

### デメリット
- 保存処理に embedding 計算と類似度検索のオーバーヘッドが追加される
- 既存データ量が増えると類似度検索のコストが増加する（将来的にインデックス最適化が必要になる可能性）
- 閾値 0.95 が全てのケースに最適とは限らない（高すぎると重複が漏れ、低すぎると異なる内容がスキップされる）
