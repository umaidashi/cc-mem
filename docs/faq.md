# FAQ

cc-mem に関するよくある質問と回答です。

---

## 基本的な質問

### Q: CLAUDE.md と cc-mem の違いは？

**CLAUDE.md** はプロジェクトのルールや指示を書く静的なファイルです。内容は手動で管理し、セッション開始時に毎回読み込まれます。

**cc-mem** は会話の履歴を自動保存し、必要なときに検索で呼び出す動的な長期記憶です。

| | CLAUDE.md | cc-mem |
|---|---|---|
| 目的 | プロジェクトルール・指示 | 過去の会話の記憶 |
| 更新方法 | 手動 | 自動 (Hook) |
| 参照方法 | セッション開始時に自動読込 | `cc-mem search` で明示的に検索 |
| データ量 | 小〜中 (1ファイル) | 大 (全セッション蓄積) |
| 用途の例 | コーディング規約、使用技術 | 設計判断の経緯、過去の議論 |

両者は補完関係にあります。CLAUDE.md に「過去の議論は cc-mem で検索せよ」と書くことで連携します。

### Q: データはどこに保存される？

```
~/.cc-mem/memory.db
```

ホームディレクトリ直下の `.cc-mem` ディレクトリに SQLite データベースとして保存されます。プロジェクトごとに分かれず、全プロジェクトの会話が1つの DB に集約されます。

### Q: クラウドにデータは送信される？

**されません。** cc-mem は完全にローカル完結です。

- データベース: ローカルの SQLite ファイル
- embedding: ローカルで動作する Ollama
- ネットワーク通信: Ollama API (`localhost`) のみ

外部サーバーへのデータ送信は一切ありません。

## 容量・パフォーマンス

### Q: DB の容量はどのくらいになる？

目安として:

| 使用量 | 会話量の目安 |
|---|---|
| 1 MB | 数十セッション |
| 10 MB | 数百セッション |
| 100 MB | 1000+ セッション |

embedding ベクトル (1チャンクあたり約3KB) が容量の大部分を占めます。`cc-mem stats` で現在の DB サイズを確認できます。

```bash
cc-mem stats
```

### Q: 古いデータを削除できる？

現時点では CLI からの削除機能は提供していません。完全にリセットしたい場合は DB ファイルを削除してください:

```bash
rm ~/.cc-mem/memory.db
rm -f ~/.cc-mem/memory.db-wal
rm -f ~/.cc-mem/memory.db-shm
```

特定のデータだけ削除したい場合は、SQLite を直接操作できます:

```bash
# 60日以上前のデータを削除する例
sqlite3 ~/.cc-mem/memory.db \
  "DELETE FROM memories WHERE created_at < datetime('now', '-60 days');"
```

### Q: 検索が遅くなることはある？

ベクトル検索は全ての embedding に対してコサイン類似度を計算するため、データ量に比例して遅くなります。ただし、数千件程度であれば体感できるほどの遅延にはなりません。

FTS5 検索は trigram インデックスを使用しているため、データ量が増えても高速です。

## Ollama 関連

### Q: Ollama なしでも使える？

**使えます。** Ollama が未起動の場合:

- **保存:** embedding なし (`null`) で保存される
- **検索:** FTS5 全文検索（trigram）のみが機能する

ベクトル検索（意味的類似度）は使えませんが、キーワードベースの検索は問題なく動作します。

### Q: nomic-embed-text 以外のモデルは使える？

環境変数 `CC_MEM_EMBED_MODEL` で変更できます:

```bash
export CC_MEM_EMBED_MODEL="mxbai-embed-large"
ollama pull mxbai-embed-large
```

ただし、モデルを変更すると既存のベクトルとの次元数が異なる場合があり、検索精度が低下する可能性があります。モデルを変更する場合は DB をリセットすることを推奨します。

### Q: Ollama を常時起動しておく必要がある？

保存時と検索時に Ollama が起動していれば十分です。macOS で Ollama.app をインストールしている場合は、ログイン時に自動起動する設定が可能です。

## Hook 関連

### Q: 全てのセッションが保存される？

Stop Hook は Claude Code がレスポンス生成を完了するたびに発火します。つまり、1つのセッション中で複数回 save が実行される可能性があります。

ただし、同一セッション内で重複するチャンクが保存されることがあります（現バージョンでは重複排除は行っていません）。

### Q: 特定のプロジェクトだけ保存したい場合は？

`matcher` フィールドに正規表現を設定することで、特定の条件でのみ Hook を発火させることができます。ただし、`matcher` はレスポンス内容に対するフィルタであり、プロジェクトパスによるフィルタは現時点ではサポートされていません。

全プロジェクトで保存し、検索時にキーワードで絞り込むのが推奨の運用方法です。

### Q: Hook の実行でClaude Codeが遅くなる？

コマンド末尾の `&` によりバックグラウンドで実行されるため、Claude Code のレスポンスには影響しません。`2>/dev/null` により stderr も抑制されます。

## 検索関連

### Q: 検索スコアの意味は？

検索結果に表示されるスコアは、以下の要素を統合した値です:

1. **FTS5 ランク** -- キーワードの一致度（BM25）
2. **ベクトル類似度** -- 意味的な類似度（コサイン類似度）
3. **RRF 統合** -- 上記2つのランクを統合
4. **時間減衰** -- 新しい記憶ほどスコアが高い（半減期30日）

スコアの絶対値に意味はなく、相対的な順位で判断してください。

### Q: 英語の会話も検索できる？

はい。FTS5 trigram トークナイザは言語に依存しないため、日本語・英語・その他の言語が混在していても検索可能です。ベクトル検索も `nomic-embed-text` が多言語に対応しています。
